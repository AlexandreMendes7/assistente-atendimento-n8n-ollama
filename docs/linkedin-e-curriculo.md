## 📄 Descrição para currículo

### Versão curta (uma linha)

> **Recepcionista virtual de IA para clínica odontológica (n8n + Ollama):** agente de IA local que responde dúvidas de pacientes (horário, tratamentos, valores, agendamento) usando LLM (Llama 3.1) sem custo de API, com engenharia de prompt e integração via webhook.

### Versão para seção de Projetos

> **OdontoFlow — Recepcionista Virtual de IA para Clínicas (n8n + Ollama)**
> Desenvolvi um agente de IA que automatiza o atendimento de clínicas odontológicas, respondendo dúvidas frequentes (horário, tratamentos, valores, convênios, localização e agendamento) de forma automática e profissional.
> - Orquestração do fluxo em **n8n** (Webhook, tratamento de contexto e resposta HTTP).
> - Modelo de linguagem **Llama 3.1 (8B)** executado localmente via **Ollama**, sem APIs pagas nem dependência de nuvem.
> - **Engenharia de prompt** (system prompt + base de conhecimento controlada) para respostas fiéis e sem alucinações.
> - Solução de **custo zero**, offline e facilmente adaptável a qualquer empresa.
> - **Tecnologias:** n8n, Ollama, Llama 3.1, REST API, JSON, engenharia de prompt.

### Bullet points (para colar em "Experiência" ou "Projetos")

- Automatizei o atendimento ao cliente de pequenos negócios com um agente de IA local (n8n + Ollama), eliminando custos de API.
- Implementei integração via webhook e chamada REST a um LLM (Llama 3.1) rodando 100% offline.
- Apliquei engenharia de prompt para garantir respostas baseadas apenas na base de conhecimento da empresa.
- Documentei e empacotei o projeto para portfólio, com workflow importável e testes reproduzíveis.
